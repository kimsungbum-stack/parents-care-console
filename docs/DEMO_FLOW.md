# 데모 시연 순서

## 1. 대시보드
- `/` 화면에서 상태별 리드 수, 다가오는 다음 연락일, 최근 상담 기록을 먼저 보여줍니다.
- 운영자가 하루 업무를 시작할 때 어떤 화면을 기준으로 우선순위를 잡는지 설명하기 좋습니다.
- 상단 버튼으로 `리드 목록`, `신규 리드 등록`으로 바로 이동할 수 있습니다.

## 2. 리드 목록
- `/leads`로 이동해 현재 운영 중인 리드가 목록으로 정리되는 모습을 보여줍니다.
- 검색과 상태 필터를 짧게 시연해도 흐름이 자연스럽습니다.

## 3. 신규 리드 등록
- `/leads/new`에서 신규 보호자 리드를 등록합니다.
- 아래 예시로 입력하면 시연이 자연스럽습니다.
  - 보호자명: `김민정`
  - 연락처: `010-1234-5678`
  - 보호자와 부모님의 관계: `딸`
  - 케어 대상 이름 또는 호칭: `부친`
  - 케어 대상 연령대: `70대 후반`
  - 현재 상황 요약: `외래 일정이 잦아 보호자 동행 부담이 커진 상황`
  - 유입경로: `네이버 검색`
  - 병원명: `서울중앙병원`
  - 진료과 정보 또는 수: `정형외과, 신경과`
  - 핵심 문제: `병원동행 부담, 일정 정리 필요`

## 4. 등록 직후 상세 화면
- 저장 직후 `/leads/[leadId]` 상세 화면으로 이동하는 흐름을 보여줍니다.
- 상단 기본 정보, 현재 상태, 다음 연락일이 바로 이어지는 점을 설명합니다.

## 5. 상태 변경과 다음 연락일 저장
- 상세 상단의 운영 상태 관리 영역에서 상태를 바꾸고 다음 연락일을 지정합니다.
- 저장 직후 상단 배지와 일정이 바로 갱신되는 흐름을 보여줍니다.

## 6. 보호자 요약 리포트 저장
- `보호자 요약 리포트` 탭으로 이동합니다.
- `현재 상황`, `이번 주 해야 할 일`, `다음 액션` 정도만 수정해서 저장하면 충분합니다.
- 읽기 모드로 돌아오면서 값이 바로 반영되는 점을 보여줍니다.

## 7. 메모와 상담 기록 저장
- `파트너/운영자 메모` 탭에서 운영 메모 1건을 추가합니다.
- `상담 기록` 탭에서 전화 상담 기록 1건을 추가합니다.
- 두 항목 모두 저장 직후 리스트 상단에 즉시 반영되는 점을 보여줍니다.

## 8. 다시 목록 확인
- `/leads`로 돌아가 방금 등록한 리드가 목록에 보이는지 확인합니다.
- 검색과 상태 필터를 짧게 다시 보여주며 전체 운영 흐름을 마무리합니다.

# Phase 3A: 통화 녹음 업로드 + AI 자동 케이스 등록

> 센터장이 보호자와 통화 후, 녹음 파일 하나만 올리면 AI가 상담 내용을 분석해서 케이스를 자동으로 만들어주는 기능.

---

## 1. 사용자 시나리오

센터장(50~60대, 비개발자)의 실제 사용 흐름:

```
1) 보호자와 전화 통화 종료
2) 앱(웹) 열기
3) 대시보드에서 "통화 녹음 올리기" 버튼 클릭
4) 녹음 파일 선택 (mp3 / m4a / wav / webm, 최대 25MB)
5) "올리기" 버튼 터치
6) 로딩 화면: "상담 내용을 정리하고 있어요..."
   - 예상 소요: 30초~2분 (녹음 길이에 따라)
7) 완료 화면: "새 케이스가 등록됐어요!"
   - AI가 추출한 핵심 정보 미리보기 카드 표시
   - "케이스 확인하기" 버튼
8) 케이스 상세 화면으로 이동 → 필요시 수정
```

### 화면 흐름 요약

| 단계 | 화면 | 주요 요소 |
|------|------|-----------|
| 진입 | 대시보드 | "통화 녹음 올리기" 큰 버튼 (항상 눈에 보이는 위치) |
| 파일 선택 | 업로드 모달 | 파일 선택 영역 + "올리기" 버튼 |
| 처리 중 | 로딩 화면 | 진행 상태 텍스트 + 스피너 |
| 완료 | 결과 카드 | 추출 정보 요약 + "케이스 확인하기" 버튼 |
| 확인 | 케이스 상세 | 자동 생성된 케이스 (수정 가능) |

## 2. 사용성 검토 (50~60대 센터장 기준)

### 직관성 체크리스트

| 항목 | 판단 | 설명 |
|------|------|------|
| 버튼 찾기 쉬운가? | 통과 | 대시보드 상단에 큰 버튼으로 배치, 색상 강조 |
| 전체 단계 수 | 통과 | 3단계 (버튼 클릭 → 파일 선택 → 올리기). 최소한의 조작 |
| 기다리는 동안 불안하지 않은가? | 주의 | 로딩 시간이 길 수 있음. 단계별 진행 텍스트 표시 필수 |
| 결과를 이해할 수 있는가? | 통과 | 전문 용어 없이 한국어로 핵심만 카드에 표시 |
| 실수했을 때 되돌릴 수 있는가? | 통과 | 케이스 생성 후 수정/삭제 가능 |

### 설계 원칙

- 버튼 텍스트는 "통화 녹음 올리기"처럼 행동을 직접 설명
- 로딩 중 텍스트는 친근한 말투: "상담 내용을 정리하고 있어요..."
- 에러 메시지도 쉬운 말로: "녹음이 너무 짧아요. 10초 이상인 파일을 올려주세요."
- 한 화면에 하나의 행동만 요구 (선택지 최소화)
- 터치 대상 크기: 최소 48px (모바일 사용 고려)

## 3. AI 분석 추출 정보

녹음 파일에서 AI가 자동으로 추출하는 정보:

| 필드 | 설명 | 예시 | 매핑 테이블 필드 |
|------|------|------|-----------------|
| 보호자명 | 전화한 사람 이름 | "김영희" | `leads.guardian_name` |
| 연락처 | 통화 중 언급된 번호 (없으면 null) | "010-1234-5678" | `leads.phone` |
| 관계 | 보호자와 케어대상의 관계 | "딸" | `leads.guardian_relationship` |
| 케어대상 이름 | 돌봄 받을 분 이름 | "박순자" | `leads.care_recipient_name` |
| 케어대상 연령대 | 나이 또는 연령대 | "80대 초반" | `leads.care_recipient_age_group` |
| 현재 상황 | 케어대상의 현재 건강/생활 상황 | "치매 초기, 혼자 거주" | `leads.current_situation_summary` |
| 긴급도 | 연락 시급성 3단계 | 아래 참조 | `leads.is_high_risk` + 별도 필드 |
| 핵심 니즈 | 보호자가 원하는 것 | ["주간보호 이용 문의", "비용 상담"] | `leads.key_issues` |
| 다음 연락 추천일 | AI가 판단한 후속 연락 시점 | "2026-03-26" | `leads.next_contact_date` |
| 추천 행동 | 센터장이 해야 할 다음 행동 | "방문 상담 일정 잡기" | 상담 메모에 기록 |

### 긴급도 분류

| 등급 | 표시 | 기준 |
|------|------|------|
| 빨리 연락 필요 | 빨간색 원 | 즉시 도움 필요, 위기 상황 언급, 다른 기관 알아보는 중 |
| 이번 주 내 연락 | 노란색 원 | 구체적 니즈 있음, 비교적 빠른 결정 원함 |
| 여유 있음 | 초록색 원 | 정보 수집 단계, 아직 결정 전 |

### AI 프롬프트 출력 형식

기존 `ConsultationAnalysis` 타입 (`src/types/ai.ts`) 구조를 확장:

```typescript
{
  guardian: {
    name: string;          // 보호자명
    phone: string | null;  // 연락처
    relation: string | null; // 관계
  };
  care_target: {
    name: string | null;       // 케어대상 이름
    age_group: string | null;  // 연령대
    condition: string | null;  // 현재 상황
  };
  consultation: {
    summary: string;           // 상담 요약
    urgency: "높음" | "보통" | "낮음";  // 긴급도
    key_needs: string[];       // 핵심 니즈 목록
  };
  recommendation: {
    next_contact: string | null;  // 다음 연락 추천일 (YYYY-MM-DD)
    next_action: string;          // 추천 행동
    pipeline_stage: LeadStatus;   // 케이스 초기 상태 (보통 "신규")
  };
}
```

## 4. 기술 흐름

```
[센터장 브라우저]
    │
    ▼
(1) 파일 업로드 (POST /api/upload-recording)
    │
    ▼
(2) Supabase Storage에 녹음 파일 저장
    │  버킷: recordings/{org_id}/{timestamp}_{filename}
    │
    ▼
(3) OpenAI Whisper API로 음성→텍스트 변환
    │  모델: whisper-1
    │  언어: ko (한국어 지정)
    │
    ▼
(4) Claude API로 텍스트 분석
    │  입력: Whisper 변환 텍스트
    │  출력: ConsultationAnalysis JSON
    │
    ▼
(5) leads 테이블에 새 케이스 자동 저장
    │  status: "신규"
    │  source: "통화녹음"
    │
    ▼
(6) consultations 테이블에 첫 상담 기록 저장
    │  channel: "전화"
    │  summary: AI 분석 요약
    │  details: Whisper 전문 텍스트
    │
    ▼
(7) 클라이언트에 새 케이스 ID 반환
    │
    ▼
[케이스 상세 화면으로 이동]
```

### API 엔드포인트

```
POST /api/upload-recording
  - Content-Type: multipart/form-data
  - Body: { file: File }
  - Response: { leadId: string, analysis: ConsultationAnalysis }
```

### Supabase Storage 설정

- 버킷명: `recordings`
- 경로 패턴: `{org_id}/{timestamp}_{original_filename}`
- 허용 MIME: `audio/mpeg`, `audio/mp4`, `audio/wav`, `audio/webm`
- 최대 파일 크기: 25MB
- RLS: 같은 org 소속만 읽기/쓰기

## 5. 엣지 케이스 및 에러 처리

| 상황 | 감지 방법 | 사용자에게 보여줄 메시지 | 기술 처리 |
|------|-----------|------------------------|-----------|
| 녹음이 너무 짧음 (10초 미만) | 파일 duration 체크 또는 Whisper 결과가 1~2단어 | "녹음이 너무 짧아요. 10초 이상인 파일을 올려주세요." | 업로드 중단, Storage에서 삭제 |
| 지원하지 않는 파일 형식 | MIME 타입 체크 | "이 파일은 올릴 수 없어요. mp3, m4a, wav, webm 파일만 가능해요." | 업로드 차단 (클라이언트 단) |
| 파일 크기 초과 (25MB 이상) | 파일 크기 체크 | "파일이 너무 커요. 25MB 이하 파일을 올려주세요." | 업로드 차단 (클라이언트 단) |
| Whisper 음성 인식 실패 | Whisper API 에러 또는 빈 텍스트 반환 | "녹음 내용을 알아듣지 못했어요. 주변이 조용한 곳에서 다시 녹음해 주세요." | Storage 파일 유지, 재시도 가능 |
| Claude AI 분석 실패 | Claude API 에러 또는 JSON 파싱 실패 | "상담 내용 정리에 실패했어요. 다시 시도해 주세요." | 텍스트는 보존, "다시 분석하기" 버튼 제공 |
| 네트워크 끊김 (업로드 중) | fetch 에러 | "인터넷 연결을 확인하고 다시 시도해 주세요." | 자동 재시도 없음, 수동 재시도 버튼 |
| 이름/연락처 추출 실패 | AI 결과에 null 값 | 케이스는 생성하되 빈 필드로 두고 "정보가 부족해요. 직접 입력해 주세요." 안내 | 필수 필드 빈값 허용, 수정 유도 |
| Whisper 할당량 초과 | API rate limit 에러 | "지금 요청이 많아요. 잠시 후 다시 시도해 주세요." | 큐잉 또는 재시도 안내 |

### 에러 복구 전략

- **파일은 항상 먼저 저장**: Supabase Storage에 업로드 성공 후 분석 시작. 분석 실패해도 파일은 남아있음.
- **부분 성공 허용**: Whisper 성공 + Claude 실패 시, 텍스트만으로 케이스 생성하고 센터장이 직접 정보 입력.
- **재시도 가능**: 분석 실패한 녹음은 "다시 분석하기" 버튼으로 재시도 가능.

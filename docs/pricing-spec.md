# Phase 4: 요금제 구조 + 온보딩 + 전환 유도

> 센터장이 무료로 시작해서 가치를 체감한 뒤, 자연스럽게 유료 플랜으로 전환하는 구조.

## 1. 요금제 구조

### 플랜 비교

| 플랜 | 가격 | 리드 한도 | 사용자 수 | 주요 기능 |
|------|------|-----------|-----------|-----------|
| 무료 | 0원 | 월 15건 | 1명 | 모든 기본 기능 |
| 스탠다드 | 월 4.9만원 | 무제한 | 3명 | 알림톡, 리포트, 내보내기 |
| 프리미엄 | 월 9.9만원 | 무제한 | 무제한 | AI 기능, 다지점, 전담 지원 |

### 핵심 가치 제안

**"어르신 1명 추가 등록 = 월 100~130만원. 구독료 4.9만원의 26배 가치."**

문의 1건이라도 놓치지 않으면 구독료는 즉시 회수된다는 메시지를 전달.

### 플랜별 포지셔닝

- **무료**: 써보고 판단하세요. 제한 없이 모든 기본 기능 체험.
- **스탠다드**: 문의가 꾸준히 들어오는 센터. 팀원과 함께 쓰고, 놓치는 문의 없이.
- **프리미엄**: 여러 지점 운영하거나 AI 자동화가 필요한 센터.

## 3. 온보딩 흐름

### 시나리오

센터장(50~60대, 비개발자)이 처음 가입했을 때의 경험:

```
1) 회원가입 완료
2) 환영 화면 표시
   - "환영합니다! 3가지만 해보면 바로 쓸 수 있어요."
3) 3단계 가이드 시작
4) 가이드 완료
5) 정상 대시보드로 전환
```

### 3단계 가이드

| 단계 | 할 일 | 유도 방법 | 완료 조건 |
|------|-------|-----------|-----------|
| 1단계 | 첫 케이스 등록하기 | "신규 케이스" 버튼을 깜빡이는 강조 표시 + 안내 툴팁 | 케이스 1건 저장 |
| 2단계 | 다음 연락일 설정하기 | 케이스 상세에서 "다음 연락일" 필드 강조 + "잊지 않게 알려드릴게요" 안내 | 연락일 1회 설정 |
| 3단계 | 파이프라인 확인하기 | 좌측 메뉴 "파이프라인" 강조 + "여기서 전체 현황을 볼 수 있어요" 안내 | 파이프라인 페이지 방문 |

### 완료 메시지

```
"잘하셨어요! 이제 문의가 올 때마다 여기에 등록하면 돼요."
```

### 화면 흐름 요약

| 단계 | 화면 | 주요 요소 |
|------|------|-----------|
| 진입 | 환영 화면 | 인사말 + "3가지만 해보면 바로 쓸 수 있어요" |
| 1단계 | 대시보드 | "신규 케이스" 버튼 강조 + 말풍선 안내 |
| 2단계 | 케이스 상세 | "다음 연락일" 필드 강조 + 말풍선 안내 |
| 3단계 | 파이프라인 | 보드 전체 + "여기서 한눈에 볼 수 있어요" 안내 |
| 완료 | 대시보드 | 축하 메시지 + 일반 대시보드 전환 |

### 설계 원칙

- 한 번에 하나만 요구. 동시에 여러 행동을 안내하지 않음.
- "건너뛰기" 버튼 항상 제공. 강제하지 않음.
- 말풍선/툴팁은 화살표로 정확한 위치를 가리킴.
- 안내 텍스트는 존댓말, 친근한 말투.

## 4. 무료 한도 전환 유도

### 단계별 메시지

| 사용량 | 조건 | 표시 위치 | 메시지 |
|--------|------|-----------|--------|
| 0~11건 | 80% 미만 | 대시보드 상단 (작은 텍스트) | "이번 달 {n}/15건 등록" |
| 12건 | 80% 도달 | 대시보드 상단 (노란색 배너) | "이번 달 남은 등록 3건이에요. 스탠다드로 업그레이드하면 걱정 없어요." |
| 13건 | | 대시보드 상단 (노란색 배너) | "이번 달 남은 등록 2건이에요." |
| 14건 | | 대시보드 상단 (주황색 배너) | "이번 달 남은 등록 1건이에요." |
| 15건 | 100% 도달 | 모달 + 등록 버튼 비활성화 | "이번 달 무료 등록을 다 쓰셨어요. 스탠다드로 올리면 걱정 없이 쓸 수 있어요." |

### 전환 유도 원칙

- **압박하지 않는다**: "한도 초과"가 아니라 "다 쓰셨어요". 부정적 느낌 최소화.
- **가치를 먼저 보여준다**: 업그레이드 버튼 옆에 "어르신 1명 = 월 100만원 이상" 문구.
- **기존 데이터는 유지**: 한도에 도달해도 기존 케이스 조회/수정은 정상 동작. 새 등록만 제한.
- **다음 달 자동 리셋**: 매월 1일 무료 한도 초기화. "다음 달에 다시 15건 쓸 수 있어요" 안내.

### 업그레이드 동선

```
1) 배너/모달에서 "업그레이드" 버튼 클릭
2) 요금제 비교 페이지 표시
   - 현재 플랜(무료) 강조
   - 스탠다드 플랜에 "추천" 뱃지
3) "스탠다드 시작하기" 클릭
4) 결제 정보 입력 (추후 결제 시스템 연동)
5) 즉시 업그레이드 적용, 리드 한도 해제
```

## 5. 사용성 검토 (50~60대 센터장 기준)

### 직관성 체크리스트

| 항목 | 판단 | 설명 |
|------|------|------|
| 온보딩이 부담스럽지 않은가? | 통과 | 3단계만, 각 단계 1클릭. 건너뛰기 가능 |
| 한도 안내가 이해되는가? | 통과 | "n/15건"이라는 직관적 표현. 복잡한 설명 없음 |
| 업그레이드가 강제로 느껴지지 않는가? | 통과 | 한도 도달해도 기존 기능 유지. "올리면 걱정 없어요" 톤 |
| 요금제 차이가 한눈에 보이는가? | 통과 | 기능 비교표에서 O/X로 명확히 구분 |
| 가격이 합리적으로 느껴지는가? | 통과 | "어르신 1명 = 26배 가치" 메시지로 설득 |

### 클릭 수 최소화 포인트

| 동선 | 클릭 수 | 설명 |
|------|---------|------|
| 가입 → 온보딩 완료 | 5~6회 | 가입(2) + 가이드 3단계(3) + 완료 확인(1) |
| 한도 배너 → 업그레이드 | 3회 | 배너 클릭(1) + 플랜 선택(1) + 결제(1) |
| 요금제 페이지 진입 | 2회 | 설정 메뉴(1) + 요금제(1) |

### 모바일 사용성 이슈

| 이슈 | 대응 |
|------|------|
| 요금제 비교표가 모바일에서 잘릴 수 있음 | 카드형 레이아웃으로 전환 (플랜별 세로 카드) |
| 온보딩 말풍선이 화면을 가릴 수 있음 | 말풍선 위치 자동 조정 + "닫기" 버튼 |
| 한도 배너가 작은 화면에서 눈에 안 띌 수 있음 | 모바일에서는 상단 고정 배너로 표시 |
| 업그레이드 모달 내 결제 폼 입력 불편 | 큰 입력 필드 + 숫자 키패드 자동 활성화 |

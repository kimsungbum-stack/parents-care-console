# 부모안심90 운영 콘솔 PRD (MVP)

## 1. 제품 한 줄 설명
병원동행/일정정리/센터연계 상담을 위한, 보호자 리드와 상담 이력을 관리하는 내부 운영 웹툴.

## 2. 문제 정의
- 상담 내용이 메신저/노트/기억에 분산되어 누락 위험이 크다.
- 리드 상태와 다음 연락일이 체계적으로 관리되지 않아 후속 대응이 늦어진다.
- 보호자에게 전달할 요약 내용이 매번 수작업으로 흩어져 재작성된다.
- 파트너/운영자 메모가 한곳에 모이지 않아 운영 맥락이 끊긴다.

## 3. 핵심 사용자
- 1차 사용자: 대표 1명 (내부 운영자)
- 사용 환경: 웹 기반(데스크톱 중심), 고객용 앱 아님

## 4. MVP 목표
- 리드를 빠르게 등록하고 상태를 표준 단계로 관리한다.
- 상담 기록과 운영 메모를 리드 단위로 누적 저장한다.
- 다음 연락일을 추적해 후속 액션 누락을 줄인다.
- 보호자용 요약 리포트를 리드별로 작성/갱신한다.

## 5. 핵심 기능 목록
1. 리드 등록
2. 리드 목록/검색/필터
3. 상담 기록 저장
4. 상태 관리  
   - 상태: `신규`, `1차답장`, `인터뷰예정`, `인터뷰완료`, `소개대기`, `보류`
5. 다음 연락일 관리
6. 보호자 요약 리포트 생성/수정
7. 파트너/운영자 메모 기록

## 6. 이번 버전에서 제외할 기능
- 결제
- 실시간 채팅
- 보호사 매칭
- 알림 푸시
- 가족 다중 계정
- 지도 추천
- AI 추천 엔진

## 7. 사용자 흐름
1. 운영자가 신규 리드를 등록한다.
2. 리드 목록에서 검색/필터로 대상 리드를 찾는다.
3. 상담 후 상담 기록을 추가하고 상태를 갱신한다.
4. 다음 연락일을 지정한다.
5. 보호자용 요약 리포트를 작성/업데이트한다.
6. 필요 시 파트너/운영자 메모를 남긴다.

## 8. 필요한 페이지 목록
1. 로그인 페이지 (`/login`)
2. 대시보드 (`/`)
3. 리드 목록 페이지 (`/leads`)
4. 리드 등록 페이지 (`/leads/new`)
5. 리드 상세 페이지 (`/leads/[leadId]`)
   - 개요 탭(기본정보/상태/다음 연락일)
   - 상담 기록 탭
   - 보호자 요약 리포트 탭
   - 파트너/운영자 메모 탭

## 9. 주요 데이터 항목
- 리드 기본정보: 보호자명, 연락처, 케어 대상 정보, 유입경로
- 리드 운영정보: 현재 상태, 상태 변경 시각, 다음 연락일
- 상담 기록: 상담 일시, 채널, 요약, 상세 메모
- 보호자 리포트: 리포트 본문, 최종본 여부, 수정 시각
- 운영 메모: 메모 유형(운영/파트너), 메모 내용, 작성 시각
- 공통 메타: 생성 시각, 수정 시각

## 10. 이후 확장 방향
- 상태 변경 히스토리 고도화(타임라인/담당자 추적)
- 연락 리마인더 자동화(이메일/캘린더 연동)
- 리포트 템플릿/자동 초안 기능
- 다중 운영자 권한 체계(역할 기반 접근 제어)
- 파트너 관리 화면 분리 및 협업 기능
